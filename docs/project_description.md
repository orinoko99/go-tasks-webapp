# Описание Проекта: Веб-Приложение для решения задач по игре Go (Baduk)

## Цели
- Веб-приложение для решения задач по Go.

## Технологии: 
- Backend - Python (FastAPI для API).
- Frontend - JS (React).
- БД - SQLite (с WAL для concurrency; мигрировать на PostgreSQL если нужно).
- SGF Парсер: Python библиотека library-sgf или badukparser для JS. Она превратит текстовый файл в дерево ходов. Можешь использовать её.
- Интерфейс доски: Используйте библиотеки, такие как WGo.js или Sabaki Board. 

## Функции/поведение:
1. Аутентификация и управление пользователями
- Регистрация и авторизация с использованием JWT‑токенов.
- Поддержка множественных пользователей с учётом конкурентного доступа к БД (concurrency control).
- Профиль пользователя: отображение статистики (количество решённых задач, текущий уровень, прогресс по сборникам).

2. Управление сборниками задач
- Список заджач и сборников задач: все файлы с задачами будут находиться в директории /problems. так же туда будут добавляться новые сборники или задачи
- Каталог сборников (из SGF‑файлов): отображение списка сборников (например, «Beginner Problems», «Tsumego Vol 1») в отдельной панели.
- Метаданные сборников: название, уровень сложности, тип сборника, (например жизнь и смерть), количество задач, описание.
- Фильтрация и поиск по сборникам (по названию, уровню сложности).

3. Работа с задачами
- Список задач в сборнике: отображение задач из выбранного SGF‑файла в виде списка в отдельной области интерфейса.
- Визуальная индикация статуса:
    -- для авторизованных пользователей — ранее решённые задачи помечаются в списке задач зелёной галочкой;
    -- новые задачи — без пометки;
    -- текущие — фон задачи в списке выделен цветом.
- Выбор задачи: при выборе из списка задача становится активной и загружается на доску с начальной позиции.
- Доступность: функция выбора задачи доступна как авторизованным, так и неавторизованным пользователям (но прогресс сохраняется только для авторизованных).

4. Игровой интерфейс
- Интерактивная доска для отображения текущей позиции задачи.
- Управление ходом: пользователь делает ход камнем соответствующего цвета.
- Обратная связь по ходу:
    -- Корректный ход: система автоматически делает ответный ход согласно решению задачи (камень другого цвета ставится на доску).
    -- Некорректный ход: отображается сообщение об ошибке, задача приостанавливается, предлагается повторить попытку или посмотреть решение.
- Навигация по задаче:
    -- «Начать с начала» — сброс позиции к начальной.
    -- «Подсказка» — показ правильного хода.
    -- «Следующая задача» — переход к следующей в списке (кнопка неактивна, если задач в этом сборнике больше нет).
    -- «Решение» — показывает решение: для каждого хода пользователя помечает зеленой точкой место куда нужно ставить следующий камень.

5. Система проверки и обратной связи
- Логика проверки:
  - Пользователь всегда играет **первым** (обычно чёрными), если в задаче не указано иное.
  - После корректного хода пользователя → автоматически ставится **ответ компьютера** (следующий ход из основной ветки SGF).
  - Если ход пользователя не ведёт ни к одной правильной ветке → ошибка, задача останавливается.
  - Допускается несколько правильных ходов (несколько веток в SGF) → система принимает любой из них.
  - Сообщение о завершении: при верном решении задачи — уведомление пользователю и автоматическая пометка задачи как решённой в списке задач в интерфейсе, а так же обновление в базе данных для текущего пользователя

6. Хранение и учёт данных
- База данных:
    -- таблица с именами и паролями пользователей; поля: user_id, user_name, password_hash.
    -- таблица user_solved_tasks для хранения информации о решённых задачах авторизованных пользователей; поля: user_id, task_id, sgf_file_name, is_solved, solved_at (дата решения).
- Хранение SGF‑файлов: централизованное хранение и парсинг SGF‑файлов для извлечения задач.

7. Дополнительные улучшения и дополнения
- Для удобства пользователя:
    -- Прогресс по сборнику: индикатор выполнения (например, «5/20 задач решено»). Эта информация берется из базы данных для конкретного пользователя.
    -- Уровень сложности: автоматическая маркировка задач по сложности, если доступно для сборника или задачи (на основе SGF файла).
    -- Таймер: опция включения таймера для тренировки скорости решения.

8. Интерфейс
- Адаптивный дизайн: корректное отображение на мобильных устройствах и ПК.
- Интерфейс доски: Используйте библиотеки, такие как WGo.js или Sabaki Board. 

## Требования к Коду
- Backend: FastAPI для REST API (/auth/register, /auth/login, /tasks/list, /tasks/solve/{id}).
- Frontend: React с маршрутами (/, /login, /tasks).
- Безопасность: Хэширование паролей (bcrypt), валидация ввода.
