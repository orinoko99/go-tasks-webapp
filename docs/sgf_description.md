Общая структура файла (EBNF, упрощённо)
Файл = коллекция из одного или нескольких GameTree:
text(; корень ; узел1 (; ветка1) (; ветка2) ; узел2 ) (; следующий GameTree — вторая задача)

( — начало дерева
; — начало узла (node)
Свойство[значение] — свойства (PropIdent — 1-2 заглавные буквы)
) — конец дерева/ветки
Пробелы/переносы игнорируются вне []
Несколько задач в одном файле = несколько (; ... ) подряд (коллекция). Идеально для библиотек задач.

Координаты (для Go)
Точки = две строчные буквы aa–ss (19×19):

1-й символ = столбец (a = левый край, s = правый)
2-й символ = ряд (a = верхний ряд, s = нижний ряд)
Примеры: aa (верхний левый), tt = пасс, pd = конкретная точка.
Сжатие: AB[aa:cc] = все точки прямоугольника aa–cc.

Свойства для задач
Задачи используют setup-режим (не обычные ходы с начала). Корень = начальная позиция.
1. Root-свойства (только в первом узле)
textFF[4]GM[1]SZ[19]  // обязательно для Го
CA[UTF-8]         // кодировка (рекомендуется)
AP[MyGoApp:1.0]   // кто создал (опционально)
2. Setup-свойства (позиция задачи, нельзя смешивать с B/W в одном узле!)

AB[pd qc ee] — добавить чёрные камни
AW[dd] — добавить белые камни
AE[ff] — очистить точки (редко)
PL[B] или PL[W] — кто ходит (обязательно для задачи!)

3. Ходы (после setup)

B[dd] — ход чёрных
W[ee] — ход белых
KO[] — разрешить запрещённый ход (ко)

4. Аннотации и разметка (для решений задач)

C[Правильный ход! Чёрные живут.] — комментарий (многострочный, экранирование \] \\)
N[1] или N[Solution] — имя узла (метка правильной ветки)
TE[] — tesuji (хороший ход)
BM[] — bad move (неправильная ветка)
CR[dd ee] — кружки на ключевых точках
SQ[ff] / TR[gg] / MA[hh] — квадрат/треугольник/X
LB[dd:A] — подпись на точке
AR[dd:ee] — стрелка

Вариации = вложенные (; ... ):

Главная ветка = правильное решение (RIGHT или аналогичные обозначения)
Боковые = неправильные (с C[Wrong] или BM[] или аналогичными обозначениями или вообще могут их не иметь)

Как выглядит SGF-файл с одной задачей
text(;FF[4]GM[1]SZ[19]CA[UTF-8]
;AB[aa cc]AW[bb]PL[B]  // setup + кто ходит
;B[dd]C[Правильный ход]N[Solution]
(;B[ee]C[Wrong move!]BM[])  // неправильная ветка
)
Как выглядит файл с несколькими задачами
Просто несколько деревьев подряд:
text(;FF[4]... ;AB[...]PL[W] ... )(;FF[4]... ;AB[...]PL[B] ... )
Приложение парсит весь файл как коллекцию — каждая задача отдельно.
Что нужно для приложения решения задач

Парсить коллекцию → для каждого GameTree:
Найти root + первый узел с setup (AB/AW/PL).
Построить доску из AB/AW.
Рекурсивно обойти дерево ходов.

Показывать текущий узел + все возможные ходы (дети).
При выборе хода — переходить по ветке, показывать C/N/CR и т.д.
Правильное решение = ветка с TE[] / N[Solution] / отсутствие BM.
Поддерживать ST (стиль вариаций) и VW (только часть доски).

5. Как из букв строятся координаты на доске

5.1. Общий принцип

Координата в SGF — это две буквы:
xy
первая буква (x) → столбец (горизонталь)
вторая буква (y) → строка (вертикаль)
Важно: отсчёт идёт с верхнего левого угла, а не снизу.

5.2. Преобразование букв в числа

Используется латинский алфавит:
a = 0
b = 1
c = 2
...
s = 18   (для доски 19x19)

5.3. Примеры

SGF	Координаты	Объяснение
aa	(0, 0)	левый верхний угол
ab	(0, 1)	на одну вниз
ba	(1, 0)	на одну вправо
dd	(3, 3)	стандартная точка
pd	(15, 3)	правее

4. Визуально (кусок доски)

   a  b  c  d
a [aa][ba][ca][da]
b [ab][bb][cb][db]
c [ac][bc][cc][dc]
d [ad][bd][cd][dd]

Формула:

(x, y) = (ord(letter1) - ord('a'), ord(letter2) - ord('a'))

5.5. Важные особенности
Нет пропуска буквы "i"
В отличие от обычной Go-нотации (где пропускают I):
В SGF:
... h, i, j ...
Пустой ход (pass)
B[]
W[]
означает пас

5.3. Размер доски влияет

Для:
SZ[9] → используются только a–i
SZ[13] → a–m
SZ[19] → a–s

5.6. Перевод в привычные координаты (если нужно)

В обычной Go-нотации:
буквы по горизонтали: A–T (без I)
числа снизу вверх
Поэтому:
SGF: dd (3,3)
→ Go: D16 (для 19x19)
Формула:
column = x
row = size - y

Главное, что нужно запомнить

SGF = (x, y) от верхнего левого угла
координата = две буквы
a = 0
первая буква → вправо
вторая → вниз

6. Советы парсеру:

Игнорируйте неизвестные свойства (но сохраняйте).
Экранирование в Text: \\ → \, \] → ].
Все свойства в одном узле уникальны.
Setup и Move — не в одном узел!
